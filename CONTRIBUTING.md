## 🎯 プロジェクトについて

このプロジェクトは、個人の学習・成長記録とナレッジ管理を目的としています。主に以下の価値を提供します：

- **学習方法論**: 効率的な学習・成長手法
- **技術知識**: プログラミング・システム設計等の技術情報
- **思考フレームワーク**: 問題解決・意思決定の手法
- **テンプレート**: 再利用可能なドキュメント構造

## 🚀 貢献の方法

### 🎯 歓迎する貢献

1. **内容の改善**
   - 誤字・脱字の修正
   - 説明の分かりやすさ向上
   - 最新情報への更新
   - 新しい視点・アイデアの追加

2. **構造の改善**
   - ドキュメント構成の最適化
   - ナビゲーション改善
   - リンク・参照の整備
   - Mermaid図表の改善

3. **新規コンテンツ**
   - 新しい学習・技術トピック
   - 実践的なテンプレート
   - ケーススタディ・事例
   - ツール・リソース情報

4. **技術的改善**
   - GitHub Pages最適化
   - マークダウン品質向上
   - 自動化・効率化提案

### 🚫 受け入れ困難な貢献

- 個人的・プライベートすぎる内容
- 商用・営利目的の宣伝
- 事実に基づかない主観的すぎる意見
- 既存コンテンツと重複する内容
- 品質が著しく低い内容

## 🐛 Issue報告

### バグ・問題報告

Issue作成時は以下のテンプレートを使用してください：

```markdown
## 🐛 問題の概要
[問題の簡潔な説明]

## 📋 詳細情報
- **ファイル**: [該当ファイル名]
- **セクション**: [該当セクション]
- **ブラウザ**: [使用ブラウザ]
- **デバイス**: [PC/スマホ等]

## 🔍 現在の状況
[現在どうなっているか]

## ✨ 期待する状況
[どうなるべきか]

## 📝 追加情報
[スクリーンショット、関連情報等]
```

### 改善提案

```markdown
## 💡 提案の概要
[提案の簡潔な説明]

## 🎯 背景・動機
[なぜこの改善が必要か]

## 📈 期待する効果
[改善による利点・価値]

## 🔧 実装アイデア
[具体的な実装方法の提案]

## 📚 参考資料
[関連リンク・事例等]
```

## 🔄 プルリクエスト

### 🎯 作成前の確認

- [ ] 既存のIssueに関連しているか確認
- [ ] 同様の変更が既に提案されていないか確認
- [ ] ドキュメントの品質基準を満たしているか確認

### 📋 プルリクエストガイドライン

1. **ブランチ命名**
   ```
   feature/topic-name
   fix/issue-description
   docs/section-improvement
   ```

2. **コミットメッセージ**
   ```
   feat: 新機能追加
   fix: バグ修正
   docs: ドキュメント更新
   style: フォーマット改善
   refactor: 構造改善
   ```

3. **プルリクエストテンプレート**
   ```markdown
   ## 📋 変更内容
   [変更の概要と詳細]

   ## 🎯 動機・背景
   [なぜこの変更が必要か]

   ## 📈 改善効果
   [この変更による利点]

   ## ✅ チェックリスト
   - [ ] 内容の正確性確認
   - [ ] リンク・参照の確認
   - [ ] フォーマット統一
   - [ ] Mermaid図表の動作確認

   ## 🔗 関連Issue
   Closes #[issue-number]
   ```

### 🔍 レビュープロセス

1. **自動チェック**: 基本的な品質確認
2. **内容レビュー**: 正確性・価値・品質の確認
3. **フィードバック**: 改善点の指摘・提案
4. **承認・マージ**: 品質基準を満たした場合

## 📝 ドキュメント改善

### 📊 品質基準

**内容の質**:
- 事実に基づく正確な情報
- 読者にとって実用的・価値ある内容
- 適切な詳細レベル（過不足なし）
- 客観的で公平な視点

**構造・形式**:
- 明確な見出し階層
- 適切な目次・ナビゲーション
- 一貫したフォーマット
- 効果的なMermaid図表活用

**表現・文体**:
- 分かりやすい表現
- 適切な敬語・丁寧語
- 統一された絵文字使用
- 読みやすいレイアウト

### 🎨 スタイルガイド

**見出し**:
```markdown
# H1: ドキュメントタイトル 🔥
## H2: メインセクション 📋
### H3: サブセクション 🔧
```

**リスト**:
```markdown
- **太字**: 重要なポイント
- `コード`: 技術用語・ファイル名
- [リンク](URL): 参考資料
```

**Mermaid図表**:
- flowchart: プロセス・手順
- mindmap: 概念・アイデア整理
- gantt: スケジュール・タイムライン
- quadrantChart: 分類・優先順位

## 🤝 コミュニティガイドライン

### 🎯 基本原則

1. **敬意と礼儀**: すべての参加者への敬意
2. **建設的な議論**: 批判ではなく改善提案
3. **学習志向**: 成長・学習を重視
4. **包括性**: 多様な視点・経験を歓迎
5. **品質重視**: 高品質なコンテンツ作成

### ✅ 推奨行動

- 💡 建設的な改善提案
- 🤝 他の貢献者への支援・感謝
- 📚 知識・経験の積極的な共有
- 🔍 継続的な学習・改善意欲
- 🌟 ポジティブなコミュニティ雰囲気作り

### 🚫 禁止行動

- 個人攻撃・誹謗中傷
- 差別的・排他的な発言
- スパム・無関係な宣伝
- 著作権侵害・盗用
- 破壊的・非建設的な行動
